# Taurus 输出文字字体一致性设计

## 目标

让 P0 新增的计算状态和错误提示与旧 UI 的正文文字使用相同字号，同时保留灰色次要信息层级。

## 现状

- `DataOutput.swift` 的指标标签和数值继承 SwiftUI 默认正文样式。
- P0 新增的“不完整选择”和“不支持组合”提示显式使用 `.caption`，因此比旧 UI 明显偏小。
- P0 新增的 Picker 占位文字由原有 `NSPopUpButton` 渲染，继续使用 AppKit 系统控件字体。
- 本地化字符串和 `—` 不单独指定字体。

## 设计

在 `DataOutput.swift` 中增加只服务于输出区域的语义字体配置：指标、数值和状态提示均声明为 `Font.TextStyle.body`。状态提示继续保留 `.secondary`，通过颜色而非缩小字号表达次要层级。

不建立全应用字体系统，不调整布局、间距、颜色或摄影机数据。

## 验证

- XCTest 断言指标和状态消息的语义字体均为 `.body`。
- 运行完整 macOS XCTest。
- 运行 Release 构建。
- 启动构建后的 App，确认状态提示与右侧指标字号一致。
